# ربات تلگرام ریموت برای Claude Code

🌐 زبان: [English](README.md) · **فارسی**

کنترل **Claude Code** روی کامپیوتر ویندوزی‌ات از هر کجا، با **تلگرام**.

یک پیام (متنی **یا** صوتی) به ربات تلگرامت بفرست، و Claude همان‌جا روی سیستمت
اجرایش می‌کند — خواندن و نوشتن فایل، اجرای دستورهای PowerShell/CMD، نصب پکیج‌های
npm، گرفتن اسکرین‌شات و خیلی چیزهای دیگر — و بعد همان‌جا در چت جواب می‌دهد. موتورش
[**Claude Agent SDK**](https://www.npmjs.com/package/@anthropic-ai/claude-agent-sdk)
رسمی است.

---

## ✨ چه کارهایی می‌کند

- 📝 پیام **متنی** و 🎙️ **صوتی** — صدا با **ElevenLabs Scribe** به متن تبدیل می‌شود.
- ⚡ **سریع** — برای هر چت یک پروسه‌ی Claude گرم نگه می‌دارد، پس هر بار هزینه‌ی راه‌اندازی سرد نمی‌پردازد.
- 💬 **پاسخ زنده** — جواب را همان‌طور که Claude می‌نویسد می‌بینی.
- 🖥️ **اسکرین‌شات و تصویر** — دستور `/screenshot` کل دسکتاپ را می‌فرستد (DPI-aware، همه‌ی مانیتورها)؛ Claude هم می‌تواند هر تصویری که می‌سازد بفرستد.
- 📎 **فایل دوطرفه** — فایل *برای* ربات بفرست تا Claude بخواندش؛ Claude هم می‌تواند هر فایلی (سند، لاگ، آرشیو، …) *برایت* بفرستد.
- 🌐 **اتوماسیون مرورگر** (اختیاری) — Claude می‌تواند یک کروم واقعی را با کنترل کامل بِراند (فرم، تب، آپلود، کلیپ‌بورد، …).
- 🧠 **حافظه** — مکالمه بین پیام‌ها ادامه دارد و حتی بعد از ری‌استارت هم باقی می‌ماند.
- 🔒 **خصوصی** — فقط چتِ تلگرامِ *خودت* پذیرفته می‌شود؛ بقیه نادیده گرفته می‌شوند.
- 🛠️ **دسترسی کامل به سیستم** — فایل‌ها، شل (PowerShell/CMD)، npm، Node.js، چند پروژه.

## 🚀 راهنمای راه‌اندازی (برای مبتدی‌ها)

این مرحله‌ها را به ترتیب انجام بده. حدود ۱۰ دقیقه طول می‌کشد. دستورها در یک ترمینال
اجرا می‌شوند — در ویندوز از **PowerShell** استفاده کن (کلید `Win` را بزن، تایپ کن
*PowerShell* و Enter).

### مرحله ۱ — نصب Node.js

این ربات روی Node.js نسخه‌ی **۲۰ یا بالاتر** اجرا می‌شود.

۱. به <https://nodejs.org> برو و نسخه‌ی **LTS** را دانلود کن.
۲. نصاب را اجرا کن و در همه‌ی مرحله‌ها **Next** را بزن.
۳. PowerShell را ببند و دوباره باز کن، بعد بررسی کن که نصب شده:

<div dir="ltr">

```bash
node -v
```

</div>

باید چیزی مثل `v20.x.x` یا بالاتر ببینی. ✅

### مرحله ۲ — گرفتن فایل‌های پروژه

اگر **Git** نصب داری:

<div dir="ltr">

```bash
git clone https://github.com/hamroi/Claude-Code-Telegram-Remote-Agent.git
cd Claude-Code-Telegram-Remote-Agent
```

</div>

Git نداری؟ صفحه‌ی ریپو را باز کن، دکمه‌ی سبز **Code → Download ZIP** را بزن،
از حالت فشرده خارجش کن، و در PowerShell با `cd` وارد آن پوشه شو.

### مرحله ۳ — نصب وابستگی‌ها

داخل پوشه‌ی پروژه اجرا کن:

<div dir="ltr">

```bash
npm install
```

</div>

این کتابخانه‌های موردنیاز ربات را دانلود می‌کند. صبر کن تا تمام شود.

### مرحله ۴ — ساختن ربات تلگرام

۱. در تلگرام **[@BotFather](https://t.me/BotFather)** را جستجو کن (نسخه‌ی رسمی با تیک آبی).
۲. دستور `/newbot` را بفرست.
۳. یک **اسم** (هرچه خواستی) و یک **یوزرنیم** (باید به `bot` ختم شود، مثل `my_claude_agent_bot`) انتخاب کن.
۴. BotFather یک **توکن** می‌دهد، شبیه `8701832864:AAG-xxxxxxxxxxxxxxxxxxxx`.
۵. آن توکن را **کپی کن** — در مرحله ۷ می‌چسبانی‌اش.

### مرحله ۵ — پیدا کردن chat ID تلگرامت

ربات فقط با **تو** حرف می‌زند که با شناسه‌ی عددی چتت شناخته می‌شوی.

۱. در تلگرام **[@userinfobot](https://t.me/userinfobot)** را باز کن و **Start** را بزن.
۲. شناسه‌ات (**Id**) را می‌دهد، عددی مثل `6474033526`.
۳. آن عدد را **کپی کن** — همان `TELEGRAM_ALLOWED_CHAT_ID` توست.

### مرحله ۶ — (اختیاری) گرفتن کلید ElevenLabs برای صدا

فقط اگر می‌خواهی **پیام صوتی** بفرستی لازم است. متن بدون آن کار می‌کند.

۱. در <https://elevenlabs.io> ثبت‌نام کن.
۲. پروفایل → **API Keys** → یک کلید بساز و کپی کن.

### مرحله ۷ — ساختن فایل `.env`

پروژه یک قالب به نام `.env.example` دارد. یک کپی به نام `.env` بساز:

<div dir="ltr">

```bash
copy .env.example .env
```

</div>

حالا `.env` را در یک ویرایشگر متن (Notepad هم کافی است) باز کن و مقادیرت را پر کن:

<div dir="ltr">

```ini
TELEGRAM_BOT_TOKEN=8701832864:AAG-xxxxxxxxxxxxxxxxxxxx   # از مرحله ۴
TELEGRAM_ALLOWED_CHAT_ID=6474033526                       # از مرحله ۵
ELEVENLABS_API_KEY=                                        # از مرحله ۶ (اگر صدا نمی‌خواهی خالی بگذار)
```

</div>

> 🔐 **`.env` را محرمانه نگه دار.** کلیدهای خصوصی‌ات در آن است. این فایل از قبل در
> `.gitignore` هست، پس **هرگز** روی گیت‌هاب آپلود نمی‌شود.

### مرحله ۸ — نصب Claude Code و ورود به حساب

Claude Code موتوری است که این ربات می‌رانَدش. **یک‌بار** روی سیستمت نصب و لاگین
می‌کنی؛ بعد ربات خودکار از همان لاگین استفاده می‌کند. *(از قبل نصب و لاگین کرده‌ای؟
برو به مرحله ۹.)*

> 💳 **حساب لازم است:** Claude Code به یک پلن پولی **Claude Pro / Max / Team /
> Enterprise** یا یک حساب **Anthropic Console** (پرداخت به‌ازای مصرف) نیاز دارد.
> پلن رایگان Claude.ai شامل Claude Code نمی‌شود.

**۸.۱ — نصبش کن**

ساده‌ترین روش (با همان Node.js مرحله ۱):

<div dir="ltr">

```bash
npm install -g @anthropic-ai/claude-code
```

</div>

> روش‌های جایگزین اگر npm را نمی‌پسندی — **یکی** از این‌ها را در PowerShell اجرا کن:
> - نصاب نیتیو (پیشنهاد Anthropic): `irm https://claude.ai/install.ps1 | iex`
> - WinGet: `winget install Anthropic.ClaudeCode`

**۸.۲ — بررسی نصب**

<div dir="ltr">

```bash
claude --version
```

</div>

باید یک شماره‌ی نسخه ببینی (مثل `2.1.x`). اگر گفت *command not found*، PowerShell را
ببند و باز کن و دوباره امتحان کن.

**۸.۳ — ورود به حساب**

Claude Code را یک‌بار اجرا کن و دستورها را دنبال کن:

<div dir="ltr">

```bash
claude
```

</div>

مرورگرت را برای ورود باز می‌کند. با حساب Claude (Pro/Max) یا Anthropic Console وارد
شو. وقتی تمام شد، به ترمینال برگرد و `/exit` را بزن. ✅ از این به بعد ربات خودکار از
همین لاگین استفاده می‌کند — نیازی به API key نیست.

> 🔑 **به‌جای لاگین، API key می‌خواهی؟** مرحله ۸.۳ را رد کن و این را به `.env` اضافه کن:
>
> <div dir="ltr">
>
> ```ini
> ANTHROPIC_API_KEY=sk-ant-...
> ```
>
> </div>
>
> کلید را از <https://console.anthropic.com> → **API Keys** بگیر.

📚 مستندات کامل نصب Claude Code: <https://code.claude.com/docs/en/setup>

### مرحله ۹ — اجرای ربات

<div dir="ltr">

```bash
npm start
```

</div>

باید در ترمینال `bot started` را ببینی. این پنجره را **باز** نگه دار — بستنش ربات را
متوقف می‌کند.

### مرحله ۱۰ — امتحانش کن!

ربات را در تلگرام باز کن و یک پیام بفرست، مثل:

> اسم کامپیوترم و ساعت فعلی چیست؟

Claude روی سیستمت اجرایش می‌کند و جواب می‌دهد. 🎉
یک ویس 🎙️ بفرست تا تبدیل صدا را تست کنی، `/screenshot` بزن تا عکسی از دسکتاپت بگیری،
یا یک فایل 📎 با کپشنی مثل *«این را خلاصه کن»* بفرست.

## 💬 دستورها

<div dir="ltr">

| Command       | What it does                                                        |
| ------------- | ------------------------------------------------------------------ |
| `/new`        | شروع مکالمه‌ی تازه (پاک‌کردن متن؛ حفظ مدل و mode)                    |
| `/model`      | تغییر مدل — `/model haiku` (سریع) · `sonnet` · `opus` (باهوش‌ترین)  |
| `/mode`       | تغییر permission mode — `plan` · `bypassPermissions` · `auto`       |
| `/screenshot` | گرفتن دسکتاپ و ارسال به‌صورت عکس (مخفف‌ها: `/ss`, `/shot`)          |
| `/browser`    | روشن/خاموش‌کردن ابزار مرورگر برای این چت — `/browser on` \| `off`   |
| `/profiles`   | لیست پروفایل‌های کروم (وقتی ابزار مرورگر فعال است)                  |
| `/profile`    | انتخاب پروفایل کروم — `/profile <name>` یا `/profile off`           |
| `/status`     | نمایش مدل، mode، پوشه‌ی کاری و وضعیت session فعلی                    |
| `/help`       | نمایش لیست دستورها (`/start` هم همین کار را می‌کند)                  |

</div>

### انتخاب مدل (سرعت در برابر هوش)

- `haiku` — سریع‌ترین، عالی برای سؤال‌های کوتاه.
- `sonnet` — متعادل (پیش‌فرض).
- `opus` — تواناترین، کندترین.

### حالت‌های دسترسی (permission modes)

- **plan** — Claude فقط نقشه می‌کشد؛ به سیستمت **دست نمی‌زند**.
- **bypassPermissions** — ابزارها (فایل، PowerShell/CMD، npm، …) بدون پرسش اجرا می‌شوند. *(پیش‌فرض — بهترین برای یک ربات ریموت)*
- **auto** — Claude برای هر اقدام خودش تصمیم می‌گیرد امن است یا نه.

> ⚠️ در `bypassPermissions`، Claude بدون تأیید روی کامپیوترت دستور اجرا می‌کند. این
> دقیقاً هدف یک ربات ریموت است — ولی ربات را فقط با خودت به اشتراک بگذار
> (`TELEGRAM_ALLOWED_CHAT_ID` همین را تضمین می‌کند).

## 🧩 ارسال و دریافت فایل

**Claude ← تو.** Claude با نوشتن یک خط مارکر در پاسخش، عکس و فایل می‌فرستد:

<div dir="ltr">

```
[[image: C:\path\to\picture.png]]     ← به‌صورت عکس تلگرام (نمایش درون‌خطی)
[[file:  C:\path\to\report.pdf]]      ← به‌صورت سند (document) تلگرام
```

</div>

ربات این‌ها را تشخیص می‌دهد، هر فایل را می‌فرستد، و مارکر را از متن حذف می‌کند.
پس می‌توانی بگویی *«یک اسکرین‌شات بگیر و بفرست»*، *«از این داده نمودار بساز و
عکسش را بفرست»*، یا *«لاگ‌ها را zip کن و فایلش را برایم بفرست»* و خودش انجام می‌شود.

**تو ← Claude.** هر فایلی (سند، عکس، ویدیو، صوت) به ربات بفرست؛ روی کامپیوترت زیر
`telegram-downloads/` ذخیره می‌شود (با `DOWNLOADS_DIR` قابل تنظیم). مسیر فایل به
Claude داده می‌شود و **کپشن** تو دستور می‌شود — مثلاً یک PDF با کپشن *«این را خلاصه کن»*.

## 🩹 رفع اشکال

**ربات به پیام‌هایم جواب نمی‌دهد.**
- مطمئن شو `npm start` هنوز در ترمینال در حال اجراست.
- بررسی کن `TELEGRAM_ALLOWED_CHAT_ID` در `.env` دقیقاً با شناسه‌ات از @userinfobot یکی باشد. هر تفاوتی به‌صورت طراحی‌شده بی‌صدا نادیده گرفته می‌شود.

**خطای «model ... not supported».**
- اندپوینت Claude تو مدل‌های متفاوتی دارد. پیام خطا مدل‌های معتبر را فهرست می‌کند —
  `src/config/index.ts` را باز کن و نقشه‌ی `MODELS` را مطابقش به‌روز کن.

**پیام‌های صوتی کاری نمی‌کنند.**
- باید `ELEVENLABS_API_KEY` در `.env` ست شده باشد. بدونش فقط متن کار می‌کند.

**اسکرین‌شات سیاه است.**
- گرفتن دسکتاپ به یک صفحه‌ی **فعال و لاگین‌شده** نیاز دارد. اگر ربات به‌صورت سرویس
  پس‌زمینه یا روی یک Remote Desktop قطع‌شده اجرا شود، تصویر ممکن است سیاه دربیاید.
  در یک نشست عادی لاگین‌شده اجرایش کن.

**ابزار مرورگر کار نمی‌کند، یا دستورها نیمه‌کاره اجرا می‌شوند.**
- در `.env` مقدار `BROWSER_ENABLED=true` بگذار، بعد برای هر چت با `/browser on` فعالش کن.
- مطمئن شو روی **`/mode bypassPermissions`** هستی — در `plan`/`auto` این‌جا Claude اقدام‌های مرورگر را کامل اجرا نمی‌کند.
- برای استفاده از پروفایل واقعی کروم، **اول همه‌ی پنجره‌های کروم را ببند**، بعد `/profile <name>`.
- بعد از تغییر هرکدام، `/new` بزن تا یک session مرورگر تازه آن‌ها را بردارد.

**پاسخ‌ها کند هستند.**
- اولین پیام بعد از اجرا کندتر است (گرم‌شدن یک‌باره). برای سریع‌ترین پاسخ از
  `/model haiku` استفاده کن. باقی تأخیر بیشتر مربوط به رفت‌وبرگشت شبکه‌ی اندپوینت Claude توست.
